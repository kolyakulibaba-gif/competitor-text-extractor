01 / Обзор

Описание продукта

Приложение принимает URL страницы сайта-конкурента, автоматически извлекает только тематический текстовый контент страницы и отображает его в отформатированном виде в области просмотра, стилизованной под лист A4.

Ключевая задача — отделить смысловой текст страницы от всего «мусора»: навигации, рекламы, баннеров, сниппетов, футеров, блоков «О нас» и прочих элементов, не относящихся к теме конкретной статьи или страницы.

Целевая аудитория

  • SEO-копирайтеры и контент-редакторы, анализирующие материалы конкурентов
  • Контент-маркетологи, исследующие структуру и подачу конкурентного контента
  • Специалисты по iGaming-контенту для мультиязычных рынков
  • Команды, работающие с автоматизированными системами рерайтинга
02 / Цели

Цели и задачи

Основные цели

  • Дать пользователю мгновенный доступ к чистому тексту любой страницы без ручного копирования
  • Сохранить структуру документа (заголовки, списки, таблицы) для точного анализа
  • Исключить весь нетематический контент автоматически, без ручной фильтрации
  • Обеспечить комфортное чтение и визуальный анализ текста в форме листа A4

Что должно быть в результате

Включать
H1–H4, параграфы, таблицы, маркированные и нумерованные списки, блочные цитаты
Исключать
Меню, хедер, футер, баннеры, реклама, CTA-кнопки, попапы, виджеты, блоки навигации, сайдбары, авторские блоки, социальные кнопки
03 / User Flow

Пользовательский сценарий

  1. 1
    Вставка ссылки Пользователь вставляет URL целевой страницы в строку ввода над областью A4.
  2. 2
    Запуск обработки Нажимает кнопку «Извлечь» или Enter. Отображается индикатор загрузки.
  3. 3
    Получение результата В области A4 появляется отформатированный текст страницы: с заголовками, списками и таблицами.
  4. 4
    Работа с текстом Пользователь читает, анализирует или копирует текст. Доступна кнопка «Копировать всё» и при необходимости — экспорт.
04 / Ядро

Алгоритм извлечения текста

4.1 Загрузка страницы

Бэкенд получает URL, загружает HTML с использованием headless-браузера (Puppeteer) или прокси-запроса. Рендеринг JS обязателен, так как большинство современных сайтов подгружают контент динамически.

4.2 Фильтрация DOM — блоки к удалению

Перед извлечением текста из DOM удаляются следующие категории элементов:

Категория HTML-элементы / CSS-классы Логика
Навигация nav, header, [role=navigation] Удалить полностью
Футер footer, [role=contentinfo] Удалить полностью
Реклама / баннеры .ad, .banner, [class*="advert"], iframe, [data-ad] Удалить полностью
Сайдбар aside, .sidebar, [role=complementary] Удалить полностью
Кнопки / CTA button, .cta, [class*="btn"], [type=submit] Удалить
Попапы / оверлеи [class*="popup"], [class*="modal"], [class*="overlay"] Удалить
Соцсети / шаринг .share, [class*="social"], [class*="follow"] Удалить
Авторский блок [class*="author"], [class*="byline"] Удалить
Форма поиска [role=search], form[action*="search"] Удалить
Комментарии #comments, [class*="comment"] Удалить
Cookie / GDPR [class*="cookie"], [id*="cookie"] Удалить

4.3 Определение основного контента

После удаления мусорных блоков применяется алгоритм выбора контентной зоны:

  • Анализ элементов article, main, [role=main], .content — приоритет в порядке перечисления
  • Если явный контейнер не найден — используется алгоритм на основе плотности текста (text density): выбирается блок с наибольшим соотношением текста к HTML-тегам
  • Резервный вариант: библиотека Readability.js (Mozilla) как fallback

4.4 Структурное сохранение

  • Теги h1–h4 конвертируются в соответствующие HTML-заголовки вывода
  • Теги ul / ol / li сохраняются с полной вложенностью
  • Тег table сохраняется со всеми ячейками, с адаптивной шириной под A4
  • Теги strong, em, blockquote сохраняются
  • Изображения, иконки, медиа — полностью удаляются
  • Гиперссылки: текст ссылок сохраняется, href удаляется (чистый текст)

Типографика

  • UI-шрифт: -apple-system / SF Pro / Helvetica Neue — системный стек Apple
  • Шрифт листа A4: Georgia или Charter — читабельный serif для контента
  • Размер тела текста в листе: 15–16px, межстрочный интервал 1.7
  • H1 в листе: 26px bold; H2: 20px bold; H3: 17px semi-bold; H4: 15px semi-bold

Принципы дизайна

Стиль — лаконичный, Apple-like. Никаких лишних декораций. Чёрный фон снижает усталость глаз при длительной работе. Белый лист контрастирует и фокусирует внимание. Зелёный — единственный цветовой акцент, для функциональных элементов.

  • Скругления: 8–12px для карточек и полей ввода
  • Тень листа A4: 0 4px 40px rgba(0,0,0,0.6)
  • Переходы: 150ms ease для hover-состояний
  • Адаптивность: лист A4 на мобильных уменьшается до 100% ширины viewport с боковыми отступами 16px

Риски и ограничения

РискВероятностьРешение

10 / Сдача

Критерии приёмки

  • При вставке URL страницы с явным тегом <article> — отображается только контент этого блока, без навигации и футера
  • Заголовки H1–H4 отображаются с визуально различимой иерархией в листе
  • Маркированные и нумерованные списки отображаются корректно, с сохранением вложенности
  • Таблицы отображаются с видимыми границами ячеек и не выходят за ширину листа A4
  • Кнопка «Копировать» копирует plain text без HTML-тегов
  • Счётчик слов и знаков корректно обновляется после каждого запроса
  • Ошибка при невалидном URL отображается в виде уведомления без краша приложения
  • Интерфейс работает корректно в Chrome 120+, Firefox 120+, Safari 17+
  • Время ответа для средней страницы (200–500 КБ HTML) не превышает 8 секунд
  • Приложение задеплоено и доступно по URL (не localhost)
11 / Сроки

Примерный план разработки

  1. 1
    Backend-ядро (3–4 дня) Node.js-сервер, Puppeteer, алгоритм фильтрации DOM, эндпоинт /api/extract, тесты на 10+ реальных URL
  2. 2
    Frontend-оболочка (2–3 дня) React-приложение, URL-строка, лист A4, типографика, тёмная тема, счётчик слов
  3. 3
    Интеграция и отладка (1–2 дня) Связка фронт + бэк, обработка ошибок, edge cases, кроссбраузерное тестирование
  4. 4
    Деплой и финальная проверка (1 день) Docker-сборка, деплой, smoke-тест на 20 реальных URL разных типов сайтов

Итого: ~7–10 рабочих дней для MVP. Функция копирования и экспорт в DOCX могут быть добавлены в следующей итерации.
